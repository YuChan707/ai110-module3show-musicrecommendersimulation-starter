.sort() vs sorted() — which to use here

.sort()	sorted()
Operates on	the list in-place	returns a new list
Original list	destroyed	preserved
Works on	lists only	any iterable
Return value	None	the sorted list
For this use case sorted() is the right choice for two reasons:

scored is built fresh each call — there is no reason to mutate it, and returning None from .sort() would mean writing two lines instead of one.
sorted() works on any iterable, including the generator expression on line 152 — .sort() cannot.
Rule of thumb: use sorted() when you need the result as a value (to assign, slice, or return). Use .sort() only when you intentionally want to sort a long list in place to avoid the memory cost of a copy.
