Core Concept
Every song gets a total score from 0.0 to 1.0. The score is a weighted sum of individual match signals. Higher score = better recommendation.

total_score = (genre_score × 0.30)
            + (mood_score  × 0.25)
            + (energy_score × 0.25)
            + (acoustic_score × 0.20)


Rule 1 — Genre Match (weight: 30%)

IF song.genre == user.favorite_genre → genre_score = 1.0
ELSE                                 → genre_score = 0.0
Binary. Genre is the strongest preference signal.
Example: user likes "pop" → Song 1 (pop) scores 1.0, Song 3 (rock) scores 0.0.


Rule 2 — Mood Match (weight: 25%)

IF song.mood == user.favorite_mood → mood_score = 1.0
ELSE                               → mood_score = 0.0
Binary. Mood is the second strongest categorical signal.
Example: user wants "happy" → Song 1 and Song 10 score 1.0, Song 8 (moody) scores 0.0.


Rule 3 — Energy Proximity (weight: 25%)

energy_diff  = abs(song.energy - user.target_energy)
energy_score = 1.0 - energy_diff
Continuous. Penalizes distance from the user's target intensity.
Example: user target = 0.8, Song 1 energy = 0.82 → 1.0 - 0.02 = 0.98. Song 6 energy = 0.28 → 1.0 - 0.52 = 0.48.

Rule 4 — Acoustic Preference (weight: 20%)

IF user.likes_acoustic == True:
    acoustic_score = song.acousticness        # reward high acousticness

IF user.likes_acoustic == False:
    acoustic_score = 1.0 - song.acousticness  # reward low acousticness (electronic)
Continuous. Rewards songs that align with the user's acoustic/electronic preference.
Example: user likes_acoustic=False, Song 5 acousticness = 0.05 → score = 0.95. Song 7 acousticness = 0.89 → score = 0.11

Build the explanation by collecting which rules fired:

if song.genre == user.favorite_genre  → "matches your favorite genre ({genre})"

Example output: "Recommended because it matches your favorite genre (pop) and matches your preferred mood (happy) and energy level is close to your target"

Why These Weights?
Signal	Weight	Reasoning
Genre (0.30)	Highest	Genre is the broadest taste boundary — wrong genre is usually a hard reject
Mood (0.25)	High	Context-dependent but very important (workout vs. study vs. relaxing)
Energy (0.25)	High	Continuous — small deviations are OK, large ones feel jarring
Acousticness (0.20)	Moderate	Important but more of a soft preference than a hard filter

✅ Algorithm Recipe for song scoring (content-based)
Fantastic progress: we’ve already mapped features and now we can turn it into a concrete scoring function for the recommender. Next I’ll give a crisp rule set that can be implemented with distance/similarity scoring.

1) Input from current song (or user preference vector)
energy, valence, tempo_bpm, danceability, acousticness
optional genre, mood (if user selects or seed song has these)
2) Normalize numeric features
energy, valence, danceability, acousticness already 0-1 (no rescale)
tempo_bpm -> scale to 0-1 via min-max from dataset (60 to 152)
tempo_n = (tempo_bpm - 60) / (152 - 60)
3) Weighted distance / similarity
Build vector:
V = [energy, valence, tempo_n, danceability, acousticness]
Choose weights (tunable):
w_energy = 0.28
w_valence = 0.28
w_tempo = 0.16
w_danceability = 0.16
w_acousticness = 0.12
Euclidean distance:
d = sqrt( sum_i w_i * (V[i] - U[i])^2 )
Similarity score:
score_content = 1 / (1 + d) or score_content = 1 - d/max_d
4) Optional categorical boost
If same genre, add +0.12
If same mood, add +0.08
(or via multiplicative 1.15/1.1)
Clip final score to [0,1]
5) Rule-based behavior (“vibe intent”)
If desired mood is “chill”: boost low energy, low tempo, high acousticness
If desired mood is “intense”: boost high energy + high tempo + low acousticness
If desired mood is “happy”: boost high valence + high danceability
"Scored song = weighted sum of content similarity + categorical boost + vibe modifiers"
6) Rank tracks
Compute final_score for every candidate except seed (or already listened)
Sort descending
Return top k (e.g. 5)
🔧 Example formula (straight to code)
`final_score = score_content

(genre==genre_seed ? 0.12 : 0)
(mood==mood_seed ? 0.08 : 0)`
